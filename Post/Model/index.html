前言#

总结一些常见的model，其实也可以将model看做是architecture。

task model#

一些关于task model的内容：

  • Computer multitasking
  • Task parallelism
  • Task (computing)

使用task model的时候，需要考虑task是什么、task的执行者是什么、task的调度者是什么：

比如Computer multitasking中，task是process或thread的执行，它的执行者是CPU，它的调度者是operating system kernel中的scheduler。

比如celery中，task就是一个一个由用户调用的task function，执行者其实就是process了，调度者其实就是它的task queue了。

关于调度者，参见：

  • Scheduling (computing)

与task model相关的另外一个概念就是：task manager，比如Task Manager (Windows)。
